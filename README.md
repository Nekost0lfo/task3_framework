# Task 3 — Config & Security (ASP.NET Core)

Учебная веб‑служба на **C# / ASP.NET Core** с аккуратными настройками запуска и базовой защитой: `csharp/src/Pr3.ConfigAndSecurity/`.

Реализовано:

- **3 источника настроек**: файл → переменные окружения → аргументы командной строки (явный порядок приоритета).
- **Ранняя проверка (валидация) настроек** и запрет запуска при ошибках.
- **Ограничение запросов из браузера**: разрешаем только доверенные `Origin`.
- **Ограничение частоты (rate limiting)**, причём **разные лимиты** для чтения и записи.
- **Защитные заголовки ответа** (минимум два).
- **Два режима**: учебный и боевой, переключение только настройкой, влияет на подробность ошибок/логов.

## Что за приложение

Это минимальная JSON API служба для работы с “элементами” (товарами/позициями) в памяти.

Маршруты:

- **`GET /api/items`** — список (read‑лимит)
- **`GET /api/items/by-id/{id}`** — элемент по id (read‑лимит)
- **`POST /api/items`** — создать элемент (write‑лимит)
- **`GET /api/mode`** — текущий режим (учебный/боевой)

Хранилище — **in‑memory** (перезапуск очищает данные).

## Как это работает (общая схема)

1) При старте приложения настройки читаются **в явном порядке приоритета**:  
   **`appsettings.json` → env (`PR3_`) → CLI args**  
2) Затем выполняется **валидация настроек**. Если есть ошибки — приложение **не стартует** и печатает причины.  
3) После запуска каждый запрос проходит через pipeline:
   - добавляются **security headers**;
   - применяется **CORS‑политика** только для доверенных `Origin`;
   - применяется **rate limiting** (разные политики на read/write маршруты);
   - ошибки возвращаются по‑разному в зависимости от режима (**учебный**/ **боевой**).

## Структура проекта и ответственность файлов

### C# (`csharp/src/Pr3.ConfigAndSecurity/`)

- **`Program.cs`**: точка входа, сбор конфигурации (файл/env/CLI), ранняя валидация, регистрация CORS/rate limit/middleware, маппинг маршрутов.
- **`appsettings.json`**: базовые значения настроек по умолчанию.
- **`Config/AppOptions.cs`, `Config/RateLimitOptions.cs`, `Config/AppMode.cs`**: типы настроек.
- **`Config/AppOptionsValidator.cs`**: правила проверки корректности (origins, лимиты и т.д.).
- **`Middlewares/SecurityHeadersMiddleware.cs`**: защитные заголовки ответа (anti‑sniff, anti‑frame, no‑cache).
- **`Middlewares/ErrorHandlingMiddleware.cs`**: единый формат ошибок + различие сообщений в режимах.
- **`Middlewares/RequestId.cs`**: `X-Request-Id` для трассировки (запись в response и логирование).
- **`Services/*`**: репозиторий элементов (in‑memory).
- **`Domain/*`**: доменные модели и DTO (запрос/ответ).

## Настройки

### Приоритет источников (явно)

1) **Файл настроек** (база)  
2) **Переменные окружения** (переопределяют файл)  
3) **Аргументы командной строки** (переопределяют всё)

### Критичные настройки и почему

- **`App:TrustedOrigins`**: определяет, кто может обращаться к API из браузера (CORS). Пустой/невалидный список — риск случайно “открыть всем” или допустить ошибку в allow‑list.
- **`App:RateLimits:*`**: защищает от завала запросами. Нулевые/отрицательные значения фактически отключают защиту.
- **`App:Mode`**: влияет на информативность ошибок. В боевом режиме сообщения намеренно менее подробны, чтобы не раскрывать детали обработки запросов.

## Запуск

### C# / ASP.NET Core

Файл настроек: `csharp/src/Pr3.ConfigAndSecurity/appsettings.json`

> Примечание: шаблон таргетит `net11.0` (preview), чтобы запускаться в окружении, где установлен только .NET 11 preview.

Запуск (по умолчанию):

```powershell
cd csharp
dotnet run --project .\src\Pr3.ConfigAndSecurity\
```

Запуск в **боевом** режиме (через env, префикс `PR3_`):

```powershell
$env:PR3_App__Mode="Боевой"
dotnet run --project .\src\Pr3.ConfigAndSecurity\
```

Запуск в **учебном** режиме через CLI (пример):

```powershell
dotnet run --project .\src\Pr3.ConfigAndSecurity\ -- --mode "Учебный"
```
